    Prerequisites¶

    Before installing Testbed as Code, ensure your environment meets these requirements.

    System Requirements¶

    Operating System¶

    Supported Platforms

    • macOS: Fully supported
    • Linux: Fully supported (Ubuntu, RHEL, CentOS, etc.)
    • Windows: ❌ Not currently supported

    Python Version¶

    • Python 3.12 or higher is required
    • We provide two setup scripts that handle Python, Ansible, and all toolchain dependencies automatically:
      • setup_uv.sh — installs UV and uses it to manage Python and dependencies (recommended)
      • setup.sh — installs pyenv and uses it to manage Python and dependencies (traditional)
    • If you prefer to manage Python yourself, we recommend UV (modern, fast) or pyenv (traditional)
    # Check if UV is installed
uv --version

# Install UV if needed
curl -LsSf https://astral.sh/uv/install.sh | sh

# Install Python 3.12
uv python install 3.12

    Benefits: 10-100x faster than pip, built-in Python management

    # Check if pyenv is installed
pyenv --version

# Install Python 3.12
pyenv install 3.12
pyenv local 3.12

    Benefits: Mature, widely used, auto-activation support

    VMware Infrastructure¶

    You need access to one of the following:

    Option 1: vCenter Server¶

    • vCenter Server 6.7 or higher
    • Administrative credentials for vCenter
    • Network connectivity from Ansible control node to vCenter API (HTTPS/443)
    • At least one ESXi host managed by vCenter
    • Datastore with sufficient space for VM deployment
    • Internet access required for downloading OVA images during deployment

    Option 2: Standalone ESXi¶

    • ESXi 6.7 or higher
    • Root or administrative credentials
    • Network connectivity from Ansible control node to ESXi (SSH/22, HTTPS/443)
    • Datastore with sufficient space
    • Internet access required for downloading OVA images during deployment

    Required Resources¶

    Minimum resources needed on your VMware infrastructure:

    Resource Minimum Recommended
    CPU 2 cores per VM 4 cores per VM
    Memory 4 GB per VM 8 GB per VM
    Disk 50 GB per VM 150 GB per VM

    Network Requirements¶

    Connectivity¶

    • Management Network: Ansible control node must reach vCenter/ESXi management interface
    • VM Network: VMs need network access based on your configuration
    • Internet Access: Required for downloading OVA files during deployment (direct or via proxy configuration)

    Ports¶

    Service Port Purpose
    vCenter HTTPS 443 vCenter API access
    ESXi SSH 22 Direct ESXi management
    ESXi HTTPS 443 ESXi API access

    Testbed as Code VM Images¶

    • Testbed as Code uses a specific OVA file based on the Ubuntu 24.04 Cloud Image with additional packages installed to support Docker containers and all packages required for automatic provisioning of settings using cloud-init.
    • This OVA will automatically be downloaded from Artifactory when executing the deploy role from the Ansible playbook.

    Access Requirements¶

    Credentials Needed¶

    Gather the following information before starting:

    Item Description Example
    vCenter Host FQDN or IP address vcenter.lab.local
    vCenter Username Admin user administrator@vsphere.local
    vCenter Password Admin password (can be encrypted)
    Datacenter Name vCenter datacenter Lab-DC
    Cluster Name vCenter cluster Cluster1
    Datastore Name Storage location shared-storage

    Security Best Practice

    Never store passwords in plain text. We'll use Ansible Vault to encrypt all credentials during setup.

    Development Tools¶

    Required¶

    • Git for version control
    • SSH client for VM access
    • IDE client (VSCode, Windsurf, etc.)

    Knowledge Prerequisites¶

    Required Knowledge¶

    • Basic understanding of YAML syntax
    • Familiarity with VMware concepts (vCenter, ESXi, VMs, datastores)
    • Basic Ansible concepts

    Verification Checklist¶

    Before proceeding to installation, verify you have:

    • macOS or Linux system
    • Access to vCenter or ESXi with admin credentials
    • Network connectivity to VMware infrastructure with Internet access
    • Sufficient resources for VM deployment
    • Git and SSH client installed

    Next Steps¶

    Once you've verified all prerequisites, continue to Installation to set up Testbed as Code.

    Need Help?

    If you're missing any prerequisites:

    • Python Setup: See our Installation Guide for detailed Python setup
    • VMware Access: Contact your infrastructure team for access
