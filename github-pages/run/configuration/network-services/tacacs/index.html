    TACACS+ Service¶

    TACACS+ AAA (Authentication, Authorization, Accounting) service using the christianbecker/tac_plus-ng Docker image. Provides centralized authentication and authorization for network devices.

    Network mode

    TACACS+ uses bridge network mode with port mapping (49:49/tcp). It does not use host network mode.

    LDAP/MAVIS backend

    The deployment templates include a commented-out MAVIS/LDAP backend block. This requires the Testbed as Code LDAP service to be deployed on the same VM first. LDAP-backed authentication is not enabled by default.

    Supported Images¶

    Image Supported Versions Notes
    christianbecker/tac_plus-ng latest (default) tac_plus-ng daemon with spawnd process manager

    Basic TACACS+ Configuration¶

    Minimal configuration to authenticate a network device with one local user:

    network_services:
  tacacs:
    hosts:
      - name: "router01"
        address: "192.0.2.1"
        secret: !vault |
          $ANSIBLE_VAULT;1.1;AES256
          ...                          # Replace with vault-encrypted shared secret
    local_users:
      - name: "tbac"
        password: !vault |
          $ANSIBLE_VAULT;1.1;AES256
          ...                          # Replace with vault-encrypted password
        member: "full_access"
    profiles:
      - name: "full_access"
        attributes:
          shell:
            priv_level: 15             # Grant privilege level 15

    Complete TACACS+ Example¶

    network_services:
  tacacs:
    service_name: "tbac_tacacs"        # Docker service/container name (default: tbac_tacacs)
    image_name: "christianbecker/tac_plus-ng"
    image_version: "latest"
    port: 49                           # TACACS+ standard port (default: 49)
    spawnd_instances_min: 1            # Minimum spawnd worker processes (default: 1)
    spawnd_instances_max: 10           # Maximum spawnd worker processes (default: 10)
    log_accounting: true               # Log accounting events (default: true)
    log_authentication: true           # Log authentication events (default: true)
    log_authorization: true            # Log authorization events (default: true)
    enable_secret: !vault |encrypted|  # Global enable password (optional, use vault)

    # NAS devices permitted to query this TACACS+ server
    hosts:
      - name: "router01"
        address: "192.0.2.1"           # IP address or CIDR of NAS device
        secret: !vault |encrypted|     # Shared secret — always use vault
        prompt: "Login: "              # Optional: custom login prompt
      - name: "switch-subnet"
        address: "192.0.2.0/24"        # CIDR to permit a subnet of NAS devices
        secret: !vault |encrypted|

    # Local user accounts for authentication
    local_users:
      - name: "tbac"
        password: !vault |encrypted|   # Login password — always use vault
        member: "full_access"          # Assign to an authorization profile
      - name: "readonly"
        password: !vault |encrypted|
        member: "read_only"

    # Authorization profiles (attribute sets assigned to users)
    profiles:
      - name: "full_access"
        attributes:
          shell:
            priv_level: 15             # Sets shell priv-lvl (Cisco IOS privilege level)
            max_priv_lvl: 15           # Sets max_priv_lvl
          custom_attributes:
            - name: "Viptela-User-Group"   # Vendor AV-pair name
              value: "netadmin"             # AV-pair value
              type: "mandatory"             # mandatory (set) or optional (add)
      - name: "read_only"
        attributes:
          shell:
            priv_level: 1
            max_priv_lvl: 1

    # Rulesets (optional: rule-based authorization)
    rulesets:
      - name: "permit_show"
        rules:
          - attribute: "cmd"
            value: "show"

    TACACS+ Field Reference¶

    Field Type Default Required Description
    service_name string tbac_tacacs No Docker service and container name
    container_name string tbac_tacacs No Docker container name override
    image_name string christianbecker/tac_plus-ng No Docker image
    image_version string latest No Docker image tag
    port integer (1-65535) 49 No TACACS+ listen port
    local_users list [] No Local TACACS+ user accounts
    hosts list [] No NAS devices permitted to query the server
    profiles list [] No Authorization attribute sets
    rulesets list [] No Authentication/authorization rule groups
    spawnd_instances_min integer (>=1) 1 No Minimum spawnd worker processes
    spawnd_instances_max integer (>=1) 10 No Maximum spawnd worker processes
    enable_secret string "" No Global enable-mode password (use Ansible Vault)
    log_accounting boolean true No Enable accounting log
    log_authentication boolean true No Enable authentication log
    log_authorization boolean true No Enable authorization log

    TACACS+ Host Fields¶

    NAS devices (hosts) are the network devices permitted to send TACACS+ queries to this server:

    Field Type Required Description
    name string Yes Label for this NAS entry (used in config only)
    address string Yes IP address or CIDR block of the NAS device
    secret string Yes Shared secret between NAS and TACACS+ server (use Ansible Vault)
    prompt string No Custom login prompt displayed to users on this NAS

    TACACS+ Local User Fields¶

    Field Type Required Description
    name string Yes Username
    password string No Login password (use Ansible Vault)
    enable_password string No Enable-mode password override
    member string No Authorization profile name (must match a profiles[].name)

    TACACS+ Profile Fields¶

    Profiles define the authorization attributes granted to users who are members of that profile:

    Field Type Required Description
    name string Yes Profile name — referenced by local_users[].member
    attributes object No AV-pair attributes for this profile

    Profile Attributes¶

    Field Type Description
    attributes.shell object Well-known shell AV-pairs
    attributes.shell.priv_level integer (0-15) Sets priv-lvl for shell authorization
    attributes.shell.max_priv_lvl integer (0-15) Sets max_priv_lvl for shell authorization
    attributes.custom_attributes list Custom/vendor AV-pairs

    Custom Attribute Fields¶

    Field Type Required Description
    name string Yes AV-pair name (e.g. Viptela-User-Group)
    value string Yes AV-pair value (e.g. netadmin)
    type string No mandatory (uses set, default) or optional (uses add)

    Deployment Details¶

    • Network mode: Bridge with port mapping 49:49/tcp (not host mode)
    • Config file: tac_plus-ng.cfg rendered to <docker_compose_dir>/tacacs/tac_plus-ng.cfg
    • Log directory: <docker_compose_dir>/tacacs/logs/
    • Verification: Deployment checks that the tbac_tacacs container reaches running status

    Relevant Files¶

    • roles/deploy/tasks/service_deployment/tacacs/configure.yaml — directory creation, config rendering
    • roles/deploy/tasks/service_deployment/tacacs/deploy.yaml — Docker image pull, container start
    • roles/deploy/tasks/service_deployment/tacacs/verify.yaml — container status check
    • roles/deploy/templates/tacacs/tac_plus-ng.cfg.j2 — tac_plus-ng daemon configuration template
    • roles/deploy/templates/tacacs/docker-compose.yml.j2 — Docker Compose definition

    Security Best Practices¶

    Always vault-encrypt secrets

    The following TACACS+ fields must always be vault-encrypted before committing: - hosts[].secret — shared secret between NAS and TACACS+ server - local_users[].password — user login passwords - enable_secret — global enable password

    ansible-vault encrypt_string 'MySharedSecret' --name 'secret'

    Restrict host addresses

    Use specific IP addresses in hosts[].address rather than 0.0.0.0/0. Each NAS device should have its own entry with the exact source IP.
