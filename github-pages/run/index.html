    Testbed as Code¶

    Automate Your Testbed Deployment¶

    Testbed as Code is a Network as Code implementation for automated VM deployment and network service configuration using Ansible and VMware infrastructure.

    The Challenge¶

    Manual VM Provisioning¶

    Hours spent clicking through vCenter UI to deploy VMs, configure networking, and install services one at a time.

    Automated Deployment

    Configuration Drift¶

    Inconsistent environments caused by undocumented manual changes, making testbeds unreproducible.

    Declarative Data Models

    Validation Gaps¶

    Deploying with missing fields, invalid IPs, or unreachable hosts — problems found only at runtime.

    Key Features¶

    Data-Driven Configuration¶

    Define your testbed using simple YAML data models. No programming required — just describe what you want, and Testbed as Code handles the rest.

    Multi-Platform Support¶

    Deploy to vCenter clusters, standalone vCenter, or individual ESXi hosts. Flexible deployment for any VMware environment.

    Schema Validation¶

    Four-tier validation ensures correctness before deployment: YAMALE schema, semantic Python rules, connectivity tests, and IP uniqueness probing.

    Network Services¶

    Automated DHCP, DNS, NTP, TFTP, HTTP, LDAP, TACACS+, and SNMP/LibreNMS deployment with Docker containers.

    Centralized Image Catalog¶

    Manage OVA/ISO images with authentication support for Artifactory, private registries, and remote servers via SCP/SFTP.

    Application Services¶

    Deploy multi-container application stacks like Netbox (DCIM/IPAM) alongside network services.

    pyATS Integration¶

    Automatic generation of pyATS testbed files for seamless test automation integration.

    Architecture¶

    Testbed as Code follows the proven Network as Code architecture with a streamlined role-based design:

    graph LR
    A[common] --> B[validate]
    B --> C[deploy]
    C --> D[operate]
    E[remove]

    Role Purpose Dependency
    common Configuration merge and testbed generation Base role
    validate Schema, semantic, connectivity, and IP validation common
    deploy VM provisioning and service deployment validate
    operate Day-2 operations (LDAP sync, password resets) deploy
    remove Safe infrastructure cleanup Roadmap

    Quick Example¶

    testbed:
  name: "my-lab"
  type: "shared"

  hypervisor:
    vcenter:
      - host: "vcenter.lab.local"
        username: "admin@vsphere.local"
        password: !vault |encrypted|
        validate_certs: false
        datacenters:
          - name: "Lab-DC"
            clusters:
              - name: "Cluster1"
                datastore: "shared-storage"
                vm_names: ["tbac_vm01"]

  compute:
    vms:
      - name: "tbac_vm01"
        deploy_from: "ova"
        ova_image_name: "tbac-base-ubuntu"
        hardware:
          cpu: 2
          memory_mb: 4096
          disk_gb: 40
        network:
          interfaces:
            - name: "ens192"
              portgroup: "VLAN-010"
              ip: "10.52.123.106"
